FU Berlin CS Consultant

A study consultant for the FU Berlin Master Informatik programme. It answers programme questions from local resources and validates proposed study plans against the deterministic 2014 Master Informatik rules.

Next.js · React · TypeScript FastAPI · Python LangGraph Exact Course Lookup Docker Compose

1What it does

A backend-first study consultant with a standalone Next.js frontend. The LLM parses and explains; deterministic Python rules decide whether a study plan is valid.

Programme Q&A

Answers rule questions from the structured rule catalogue and course-offering questions from exact JSON buckets.

Study-Plan Check

Parses a proposed plan, validates LP totals and area rules, and explains issues.

Degree Rules

Structured, read-only catalogue of programme rules served via a dedicated API.

Advisory only — official FU Berlin documents and the examination office remain authoritative for real study decisions.

CourseKeySelector handles only course-offering questions and may return zero, one, or many lookup buckets such as sose26/technical/swp. CourseLookup then returns the complete selected buckets from local JSON.

5Study-Plan Validation

The rule checker is plain Python — auditable, deterministic, and never delegated to the LLM. It currently covers:

  • 90 LP module area before thesis
  • 30 LP Masterarbeit
  • 70–80 LP Informatik area
  • 10–20 LP Anwendungsbereich
  • Specialization minimums
  • Wissenschaftliches Arbeiten min/max
  • Softwareprojekt min/max
  • Wahlbereich seminar/project caveat
  • 25–30 LP ungraded modules
  • Max 15 LP Bachelor modules
  • Duplicate module detection

6Study Plan in action

The Study Plan tab renders the latest extracted plan with LP totals, the chosen Vertiefung, Wahlbereich data, and the per-area module breakdown.

A Save as PDF button in the top-right exports the current study plan to a print-ready PDF for sharing with advisors or saving alongside official paperwork.

7LLM Providers

Three interchangeable providers, selected via LLM_PROVIDER.

AcademicCloud

Hosted API, defaulting to qwen3-235b-a22b.

FU Ollama

FU cuda01 reached automatically through an SSH tunnel.

Local Ollama

Ollama running on the host machine, reached via host.docker.internal.

8Frontend
