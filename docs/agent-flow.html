FU Berlin CS Consultant Agent Flow

Backend-only LangGraph flow. Course offerings use exact JSON bucket lookup; pure degree-rule questions go directly to the answer node, and study-plan validation is owned by deterministic Python code. Every session is bound to one degree program (degree_id seeded into the checkpoint at creation); each node resolves its prompts, parse schema, and validator from the degree registry in backend/app/domain/degrees/. The LLM never chooses the degree.

Lookup scope: active course-offering lookup reads the session degree's projection of backend/app/domain/data/course_offerings.json through keys like sose26/technical/swp. Degrees without tagged offerings get an honest "no local course-offering data" note without an LLM selector call. The legacy Qdrant retrieval and query-rewriter nodes have been removed from the runtime. degree_question messages skip lookup and are answered from the degree's RULES_CONTEXT, rendered from that degree's program_rules.py. The plan_check path runs parser, deterministic rule checker, then AnswerComposer; parser schema and validator come from the degree package (msc_informatik, msc_data_science).
WizardFlow tracing: every request receives a UUID stored as wizardflow_message_id. LLM nodes record exact llm_input payloads containing prompt and msg, plus llm_output; deterministic nodes record input and output. The __start__ graph step is not logged. __end__ is finalized as a payload-free marker. Transcript text is unredacted. JSONL traces are written under backend/traces/ and excluded from Git.
